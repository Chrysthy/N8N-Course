? Principais códigos de respostas

No n8n, os códigos de resposta que você encontra são, na grande maioria, os Status Codes do protocolo HTTP padrão.

* 1. Sucesso (2xx)
Esses códigos indicam que o node (nó) executou sua função corretamente ou que a API externa aceitou sua solicitação.

- 200 OK: O padrão ouro. A requisição funcionou perfeitamente. Se for um Webhook, significa que o workflow foi acionado e (se configurado) retornou dados.

- 201 Created: Muito comum ao usar o node HTTP Request com o método POST. Significa que o recurso (ex: um novo contato no CRM) foi criado com sucesso.

- 204 No Content: A requisição funcionou, mas não há dados para retornar. É o comportamento padrão de muitos Webhooks no n8n se você não usar um node "Respond to Webhook".


* 2. Erros do Cliente (4xx)
Estes são os erros mais comuns durante a construção do workflow. Geralmente significam que você (ou o n8n) enviou algo errado para a API.

- 400 Bad Request: A estrutura dos dados enviados está incorreta.
    Verifique o JSON que você está montando no node HTTP Request. Faltou uma vírgula? Um campo obrigatório está vazio?

- 401 Unauthorized / 403 Forbidden: Problemas de permissão.
    Suas Credentials (Credenciais) estão erradas, expiradas ou a API Key não tem permissão para acessar aquele endpoint específico.

- 404 Not Found: O recurso não existe.
    A URL no node HTTP Request está errada ou o ID do item que você tentou atualizar não existe no sistema de destino.

- 429 Too Many Requests: Você atingiu o limite de taxa (rate limit) da API.
    Isso acontece muito em loops. Solução: Use o node Split in Batches e adicione um node Wait para desacelerar o fluxo.


* 3. Erros do Servidor (5xx)
O problema está no serviço que você está tentando acessar (ou no próprio servidor do n8n se você estiver usando self-hosted).

- 500 Internal Server Error: O servidor remoto travou ou encontrou uma condição inesperada.
    Dica: Tente reenviar a requisição após alguns minutos.

- 502 Bad Gateway / 504 Gateway Timeout: O tempo de resposta excedeu o limite.
    Se o seu workflow é muito complexo e demora muito para responder a um Webhook, o serviço que chamou o Webhook pode "desistir" e gerar esse erro.



! Nota: Além dos códigos HTTP, o n8n tem seus próprios erros de execução, como JSON Parsing Error (erro de sintaxe) ou ERROR: UNKNOWN (falha interna do node), que aparecem em vermelho na interface de execução.
